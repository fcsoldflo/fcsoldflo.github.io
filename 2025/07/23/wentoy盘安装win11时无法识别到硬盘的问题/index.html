Ventoy 安装 Windows 11 无法识别硬盘问题解决方法

🧩 问题描述

我原先将 Windows 和 Arch Linux 安装在同一块硬盘中。由于 Windows 更新导致引导项被删除，系统只能进入 Arch Linux，无法访问 Windows。尝试修复无效后，我决定保留原硬盘中的 Arch 系统，并新增一块硬盘专门用于安装 Windows。

这样做的目的是防止 Windows 和 Arch 再次互相影响，保持系统的稳定性。

⚙️ 硬件配置说明

  • 新硬盘：Kioxia 凯侠 SD10（SATA 固态）
  • 处理器：Intel 第 12 至 13 代
  • 启动盘工具：Ventoy（用于引导 Windows 安装镜像）

🐛 遇到的问题

使用 Ventoy 启动 Windows 安装镜像后，在“选择安装位置”界面：

  • 无法识别任何硬盘（包括新旧硬盘）
  • 只能识别外接的 Ventoy 启动盘

但进入 BIOS 时，可以正常识别到新硬盘。

🧪 原因分析

该问题是由于安装程序缺少 Intel VMD 驱动所致，尤其常见于 12/13 代 Intel CPU 平台。

✅ 解决方案

我们需要手动加载相应的硬盘驱动文件。

第一步：准备驱动文件

  1. 下载驱动压缩包：SetupRST_extracted.zip
    提取码：2025

  2. 将压缩包解压后，把整个文件夹（不要只复制文件）放入 U 盘根目录。

第二步：加载驱动

  1. 启动进入 Windows 安装界面
  2. 在“选择安装 Windows 的位置”界面，点击左下角的 “加载驱动（Load driver）”
  3. 浏览到你 U 盘中的目录：
    Drivers/VMD/A77F/
  4. 选择驱动后，点击下一步，系统即可识别出硬盘

第三步：继续安装

此时你应该可以正常看到硬盘列表，选择目标分区安装即可。之后的安装流程不会再有异常。

📝 补充建议

  • 如果你是使用 RST 模式（非 AHCI），加载该驱动是必须的。
  • 驱动不匹配或路径错误会导致加载失败，务必确认放置位置正确。
  • Ventoy 本身不影响安装，只要镜像无问题即可。
